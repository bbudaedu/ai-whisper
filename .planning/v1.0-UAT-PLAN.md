# v1.0 End-to-End (E2E) UAT Test Plan

## 1. 測試目標
驗證 FaYin v1.0 里程碑的所有核心功能，確保使用者在 PC 與行動裝置上均能順利完成「登入 -> 提交 -> 追蹤 -> 下載」的完整生命週期。

---

## 2. 測試環境與工具
- **PC 端**: Chrome / Firefox / Edge (最新版本)
- **手機端**: iOS Safari / Android Chrome
- **測試帳號**:
    - Email/Password 測試帳號
    - 具備 Google OAuth 權限的測試帳號

---

## 3. E2E 測試路徑 (Test Scenarios)

### S-01: 身份認證流程 (Authentication)
1. **路徑**: 進入登入頁 -> 輸入 Email/Password -> 進入 Dashboard -> 登出。
2. **路徑**: 進入登入頁 -> 點擊 Google 登入 -> 完成授權 -> 進入 Dashboard。

### S-02: 任務提交與處理 (Submission & Processing)
1. **路徑 (檔案)**: Dashboard -> 點擊「提交任務」-> 選擇「檔案上傳」-> 選擇音檔 -> 設定參數 -> 提交。
2. **路徑 (YouTube)**: Dashboard -> 點擊「提交任務」-> 選擇「YouTube 連結」-> 輸入網址 -> 設定參數 -> 提交。

### S-03: 任務追蹤與歷史 (Tracking & History)
1. **路徑**: 在 Dashboard 觀察新任務狀態從 `pending` -> `running` -> `done`。
2. **路徑**: 進入「歷史記錄」-> 點擊展開任務詳情 -> 檢視事件歷程。

### S-04: 結果下載 (Download)
1. **路徑**: 在任務詳情點擊「下載全部 (Zip)」。
2. **路徑**: 在任務詳情選擇特定格式（如 TXT）下載。

### S-05: 播放清單管理 (Playlists)
1. **路徑**: 進入「播放清單」-> 新增 YouTube 清單 -> 切換追蹤開關 -> 刪除清單。

---

## 4. PC 端檢查清單 (Desktop Checklist)
- [ ] 側邊導覽列 (Sidebar) 顯示正常且連結正確。
- [ ] 檔案上傳支援拖放 (Drag & Drop) 或點擊選擇。
- [ ] 任務狀態 Badge 顏色正確且易於辨識。
- [ ] 歷史紀錄列表支援分頁或捲動載入。
- [ ] 下載按鈕點擊後能觸發瀏覽器下載對話框。

---

## 5. 手機端檢查清單 (Mobile Checklist)
- [ ] 底部導覽列 (Bottom Navigation) 顯示正常且觸控區域足夠。
- [ ] 提交表單在小螢幕上不會發生溢出 (Overflow) 或按鈕被遮擋。
- [ ] 狀態 Badge 在小螢幕上能正確換行或縮放。
- [ ] Google 登入彈窗在行動瀏覽器中能正確處理回調。
- [ ] 歷史紀錄的展開/收合操作在觸控下流暢。

---

## 6. 回歸測試 (Regression)
- [ ] 修改 TaskTracker 的輪詢頻率是否影響效能。
- [ ] 在任務執行中取消 (Cancel) 是否能正確釋放背景資源。
- [ ] 檔案名稱包含中文字元或特殊符號時下載是否亂碼。
