<task type="auto" tdd="false">
  <name>Task 1: Implement ZIP download logic with RBAC and explicit path lookup</name>
  <files>api/routers/download.py</files>
  <read_first>
    - api_server.py
    - pipeline/queue/repository.py
  </read_first>
  <action>
    Create the download router and logic:
    1. Create `api/routers/download.py`. Create `router = APIRouter(prefix="/api/tasks", tags=["Download"])`.
    2. Add `@router.get("/{task_id}/download")`.
    3. Retrieve the current user from the JWT token.
    4. Fetch the task using `TaskRepository.get_task`.
       - Enforce D-25 / D-06: If user role is `external`, verify `user_id == task.requester`. If not, return `403 Forbidden`. If user role is `internal`, allow access.
    5. If task is not `DONE`, return `400` error.
    6. Define the output file lookup strategy:
       - Since `video_id` might be empty for upload tasks, use `output/{task_id}` as the primary directory (or check `task.output_path` if available).
       - Ensure logic explicitly looks in `output/{video_id}` for youtube tasks or `output/{task_id}` for upload tasks.
       - Look for `.srt`, `.txt`, `.xlsx`, `.docx`. Return 404 if no output files found.
    7. Use `tempfile.NamedTemporaryFile` and `zipfile` to create a ZIP containing these files. Exclude original audio/video files. Name the zip `{task_id}_{timestamp}.zip`.
    8. Return the ZIP using `fastapi.responses.FileResponse`. Ensure the temporary file is deleted after sending (using `background_tasks.add_task(os.remove, temp_zip.name)`).
  </action>
  <verify>
    <automated>grep -q "@router.get(\"/{task_id}/download\")" api/routers/download.py && grep -q "zipfile" api/routers/download.py</automated>
  </verify>
  <done>Download endpoint and ZIP logic implemented with security and proper path resolution.</done>
</task>
