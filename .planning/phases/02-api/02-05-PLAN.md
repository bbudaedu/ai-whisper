<task type="auto" tdd="false">
  <name>Task 1: Implement ZIP download logic</name>
  <files>api/routers/download.py</files>
  <read_first>
    - api_server.py
    - pipeline/queue/repository.py
  </read_first>
  <action>
    Create the download router and logic:
    1. Create `api/routers/download.py`. Create `router = APIRouter(prefix="/api/tasks", tags=["Download"])`.
    2. Add `@router.get("/{task_id}/download")`.
    3. Verify token and task ownership using `TaskRepository.get_task`.
    4. If task is not `DONE`, return `400` error.
    5. Find the output directory for `task.video_id` (usually `output/{video_id}`). Look for `.srt`, `.txt`, `.xlsx`, `.docx`.
    6. Use `tempfile.NamedTemporaryFile` and `zipfile` to create a ZIP containing these files. Exclude original audio/video files. Name the zip `{task_id}_{timestamp}.zip`.
    7. Return the ZIP using `fastapi.responses.FileResponse`. Ensure the temporary file is deleted after sending (using `background_tasks.add_task(os.remove, temp_zip.name)`).
  </action>
  <verify>
    <automated>grep -q "@router.get(\"/{task_id}/download\")" api/routers/download.py && grep -q "zipfile" api/routers/download.py</automated>
  </verify>
  <acceptance_criteria>
    - `api/routers/download.py` exists and contains `@router.get("/{task_id}/download")`
    - It uses `zipfile` module to package files
    - It checks `task.status == TaskStatus.DONE`
    - It uses `FileResponse` to return the zip file
  </acceptance_criteria>
  <done>Download endpoint and ZIP logic implemented.</done>
</task>
