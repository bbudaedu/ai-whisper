<task type="auto" tdd="false">
  <name>Task 2: Integrate auth endpoints and token persistence into API server</name>
  <files>api_server.py, pipeline/queue/repository.py, api/auth.py</files>
  <read_first>
    - api_server.py
    - api/auth.py
    - api/schemas.py
  </read_first>
  <action>
    Update `api_server.py` and repository to include the new authentication routes and refresh logic:
    1. Implement repository methods in `pipeline/queue/repository.py` (or auth equivalent):
       - Hash and store refresh token in DB: `create_refresh_token(user_id: str, token_hash: str, expires_at: datetime)`
       - Verify token on refresh: `verify_and_revoke_refresh_token(token_hash: str)` (check if exists, not revoked, not expired, then mark as revoked or allow depending on rotation strategy)
       - Revoke token: `revoke_refresh_token(token_hash: str)`
    2. In `api_server.py`, import `APIKeyHeader` from `fastapi.security` and `create_access_token` from `api.auth`.
    3. Add endpoint `POST /api/auth/token` that takes an API key via header (`x-api-key`) and returns a JWT token.
       - Implement API key logic per D-07: lookup API keys from DB.
       - Generate `access_token` and a secure random `refresh_token`.
       - Hash the `refresh_token` and store it in the DB using the new repository method.
       - Return `Token` response containing both `access_token` and `refresh_token`.
    4. Create an endpoint `POST /api/auth/refresh` to use the refresh token to get a new access token.
       - Take `RefreshRequest`.
       - Hash provided token, verify against DB (ensure not revoked/expired).
       - Revoke the old refresh token (rotation) and issue a new `access_token` and new `refresh_token`, storing the new hash in DB.
    5. Create an endpoint `POST /api/auth/revoke` to explicitly revoke a refresh token.
       - Take `RevokeRequest`, hash token, mark as revoked in DB.
  </action>
  <verify>
    <automated>grep -q "@app.post(\"/api/auth/token\")" api_server.py && grep -q "@app.post(\"/api/auth/refresh\")" api_server.py</automated>

    - Token generation creates and persists refresh token hash in DB
    - Refresh endpoint validates DB hash and rotates token
    - The endpoints return JSON containing `access_token` and `refresh_token`
  </acceptance_criteria>
