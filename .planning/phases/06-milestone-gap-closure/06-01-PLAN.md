# Plan: Phase 06 - 里程碑缺口修復 (Milestone Gap Closure)

## Goal
修復 v1.0 稽核發現的整合斷鏈與文件不同步問題，確保 API 路徑、資料格式與 UI 狀態完全一致，並更新專案文件反映最新實作狀態。

## Critical Files
- `web-ui-external/src/pages/TaskHistory.tsx` (API 路徑與下載邏輯)
- `web-ui-external/src/pages/TaskTracker.tsx` (狀態枚舉與下載邏輯)
- `web-ui-external/src/pages/SubmitTask.tsx` (格式名稱對齊)
- `api/routers/download.py` (格式相容性)
- `REQUIREMENTS.md` (進度更新)
- `PROJECT.md` (進度更新)

## Proposed Changes

### Wave 1: API 路徑與格式名稱修復
- **web-ui-external/src/pages/TaskHistory.tsx**:
    - 將 `usePolling` 的 endpoint 從 `tasks/history` 改為 `api/tasks/history` (或確保 client 已處理，目前的 client baseURL 為 `/api`)。
    - 修復 `downloadUrl` 函式，確保產出的 URL 包含 `/api` 前綴。
- **web-ui-external/src/pages/TaskTracker.tsx**:
    - 將 `usePolling` 的 endpoint 從 `tasks` 改為 `api/tasks` (或對應 client baseURL)。
    - 修復 `downloadUrl` 函式，確保產出的 URL 包含 `/api` 前綴。
- **web-ui-external/src/pages/SubmitTask.tsx**:
    - 將 `docx` 改為 `word` (或在後端做對照)，確保前端與後端預期的格式標籤一致。
    - 調整提交時的 `output_formats` 處理邏輯。

### Wave 2: 狀態枚舉與下載邏輯增強
- **web-ui-external/src/pages/TaskHistory.tsx**:
    - 更新 `TaskRecord['status']` 型別，對齊後端 `TaskStatus` (queued, pending, running, done, failed, canceled)。
    - 優化 `getStatusBadge` 以對應正確的後端狀態。
- **web-ui-external/src/pages/TaskTracker.tsx**:
    - 更新 `TaskRecord['status']` 型別與 `getStatusBadge`。
- **api/routers/download.py**:
    - 在 `download_task_results` 中增加格式對照表，支援 `word` -> `.docx` 與 `excel` -> `.xlsx` 的映射，提高前端相容性。

### Wave 3: 文件與進度同步
- **REQUIREMENTS.md**:
    - 將所有已完成的 v1 需求標記為 `[x]`。
    - 更新 Traceability 表格中的 Status 為 `Complete`。
- **PROJECT.md**:
    - 將 `Active` 區塊中已完成的項目移至 `Validated`。
    - 更新 `Key Decisions` 的 Status。
- **.planning/STATE.md**:
    - 更新目前的 Phase 為 `Phase 06` 並標記進度。

## Verification Plan

### Automated Tests
- 執行現有的 API 測試確保沒有回歸：`pytest tests/test_external_api_tasks.py` (若已實作)。
- 檢查前端編譯是否正常：`cd web-ui-external && npm run build`。

### Manual Verification
1. **API 路徑驗證**:
    - 進入「歷史記錄」頁面，確認不再出現 404 錯誤。
    - 點擊「下載」按鈕，確認下載請求正確發送到 `/api/tasks/.../download`。
2. **格式下載驗證**:
    - 提交一個選擇 Word/Excel 的任務，完成後嘗試在歷史頁面下載特定格式，確認後端能正確識別 `word` 參數並回傳 `.docx` 檔案。
3. **狀態顯示驗證**:
    - 觀察任務從提交到完成的過程，確認 UI 狀態標籤（等待中、處理中、已完成等）與後端實際狀態同步且顯示正確。
