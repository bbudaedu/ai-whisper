<task type="auto">
  <name>Task 1: Extract auth endpoints to api/routers/auth.py</name>
  <files>api_server.py, api/routers/auth.py, api/auth.py</files>
  <action>
    將 `api_server.py` 中的 `/api/auth/token`, `/api/auth/refresh`, `/api/auth/revoke` 端點抽離到 `api/routers/auth.py` 中：
    - 建立 `APIRouter()` 並加上 `tags=["Auth"]`。
    - 注意將 `app.post` 改為 `router.post`，並且路由為 `/token`, `/refresh`, `/revoke` (因為會在 `api_server.py` prefix 為 `/api/auth`)。
    - 調整 `api/auth.py` 的 `refresh_token_expiry` 讓其預設為 30 天 (REFRESH_TOKEN_EXPIRE_DAYS=30)。
    - 調整 access token expiry 預設為 15 分鐘 (ACCESS_TOKEN_EXPIRE_MINUTES=15) 以符合 D-13 規範。
    - 重構 `/refresh`：依據 D-15 (強迫輪替)，原邏輯已支援 verify_and_revoke 並核發新 Token，維持此邏輯。
    - 重構 `/revoke`：依據 D-16，需修改為撤銷 "使用者" 的所有 token。原邏輯僅撤銷單一 token。因此，更新 `revoke` API 為：
      1. 從 request 的 `refresh_token` 取回 user_id。
      2. 透過 `repo.revoke_all_user_refresh_tokens(user_id)` 撤銷所有 token。
      (在此步驟一併更新 `pipeline/queue/repository.py` 以實作 `revoke_all_user_refresh_tokens(self, user_id: str)`)。
  </action>
  <verify>
    <automated>grep -E "@router.post\(\"/token\"|@router.post\(\"/refresh\"" api/routers/auth.py</automated>
  </verify>
  <acceptance_criteria>
    - `api/routers/auth.py` has `@router.post("/token"`
    - `api/routers/auth.py` has `@router.post("/refresh"`
    - `api/routers/auth.py` has `@router.post("/revoke"`
    - `pipeline/queue/repository.py` has `def revoke_all_user_refresh_tokens(self, user_id: str):`
  </acceptance_criteria>
  <done>Existing endpoints extracted and updated</done>
</task>
