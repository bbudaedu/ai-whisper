<task type="auto">
  <name>Task 2: Add Google OAuth endpoints and enforce scope/domain policies</name>
  <files>api/routers/auth.py</files>
  <action>
    在 `api/routers/auth.py` 中：
    - 匯入 `from authlib.integrations.starlette_client import OAuth`
    - 初始化 `oauth = OAuth()`
    - 註冊 Google: `oauth.register(name="google", client_id=os.environ.get("GOOGLE_CLIENT_ID"), client_secret=os.environ.get("GOOGLE_CLIENT_SECRET"), server_metadata_url="https://accounts.google.com/.well-known/openid-configuration", client_kwargs={"scope": "openid email profile"})`
    - 依據 D-11，**不得**限制 `hd` 或 domain allowlist。
    - 依據 D-12，明確**不實作** `/google/unlink` 端點。
    - 新增 `@router.get("/google/login")`: 使用 `await oauth.google.authorize_redirect(request, redirect_uri)`。 redirect_uri 可以由前端傳入或預設固定 API URL。
    - 新增 `@router.get("/google/callback")`: 使用 `token = await oauth.google.authorize_access_token(request)`，並取得 `user_info = token.get("userinfo")`。取得 email, name, avatar_url 後，呼叫 `repo.authenticate_google_user` (Task 3)。
    - 若成功，返回包含 access_token 與 refresh_token 的 JSON 或前端指定之格式。為簡單起見，可以直接返回 `Token` JSON 或 HTML。
  </action>
  <verify>
    <automated>grep -E "@router.get\(\"/google/login\"|@router.get\(\"/google/callback\"" api/routers/auth.py && ! grep "/google/unlink" api/routers/auth.py</automated>
  </verify>
  <acceptance_criteria>
    - `api/routers/auth.py` configures Authlib OAuth with Google
    - `api/routers/auth.py` has `@router.get("/google/login"`
    - `api/routers/auth.py` has `@router.get("/google/callback"`
    - No `/google/unlink` endpoint exists.
  </acceptance_criteria>
  <done>Google OAuth endpoints defined without domain limits or unlink feature</done>
</task>

<task type="auto">
  <name>Task 3: Implement Google user binding and profile sync in repository</name>
  <files>pipeline/queue/repository.py</files>
  <action>
    在 `pipeline/queue/repository.py` 中實作 `authenticate_google_user`:
    - `def authenticate_google_user(self, email: str, google_sub: str, name: str, avatar_url: str) -> Optional[User]:`
    - 依據 D-09 (同步基本資料)：無論是新建立、綁定既有、或是既有已綁定，都更新 (Sync) `User` 的 `name` 與 `avatar_url` (若 OAuth 有提供新值)。
    - 檢查是否有對應的 `Identity` (provider="google", provider_id=google_sub)。若有，回傳關聯的 `User` 並更新 profile。
    - 如果沒有 `Identity`，檢查是否有同 `email` 的 `User`。
    - 若找到同 email `User`，則建立新的 `Identity` 綁定該 user，並更新 user 的 profile。
    - 若找不到 `User`，則建立新 `User` 並建立 `Identity`，同時寫入 profile。
    - 檢查最終 `User` 的 `is_active`。若為 false，回傳 `None`。若為 true，回傳 `User`。
  </action>
